📋 PROCESSUS DE SUPPRESSION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OPTION 1: SUPPRESSION DÉFINITIVE (Hard Delete)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Conditions:
  ✓ Utilisateur a la permission
  ✓ Case "Suppression définitive" cochée
  ✓ Code de vérification correct

Actions:
  1. Suppression de tous les paiements
  2. Suppression de tous les abonnements bus
  3. Suppression de tous les abonnements cantine
  4. Suppression de l'élève
  5. Création d'un log système (corbeille)

Traçabilité:
  Table: administration_systemlog
  Action: SUPPRESSION_DEFINITIVE
  Détails: Tous les éléments supprimés sont sauvegardés

OPTION 2: SUPPRESSION DOUCE (Soft Delete)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Conditions:
  ✓ Case "Suppression définitive" NON cochée
  OU
  ✓ Utilisateur n'a PAS la permission

Actions:
  1. Changement du statut à 'EXCLU'
  2. Création d'un historique
  3. Création d'un log d'activité
  4. Toutes les données sont CONSERVÉES
